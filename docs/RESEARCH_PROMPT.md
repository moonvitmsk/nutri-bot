# NutriBot Moonvit - Deep Research Prompt

## Для чего этот документ

Это полное описание проекта NutriBot для глубокого исследования и анализа. На основании этого документа проведи аудит функциональности, UX, архитектуры и предложи конкретные улучшения.

---

## 1. Что такое NutriBot

AI-бот-нутрициолог в мессенджере MAX (российский мессенджер от VK). Бот:
- Консультирует по питанию через свободный текстовый чат (GPT-4.1-mini)
- Распознаёт фото еды и считает КБЖУ (GPT-4.1 Vision API)
- Анализирует фото результатов анализов крови
- Проводит глубокие консультации (4 параллельных AI-агента: диетолог, здоровье, лайфстайл, составитель отчётов)
- Декодирует QR-коды с крышек витаминов Moonvit для активации Premium-подписки
- Ведёт дневник питания с трекингом воды и streak

**Заказчик:** Moonvit (ООО "МУНВИТ") - производитель витаминных комплексов
**CEO:** Александр Герастовский
**Платформа:** MAX (мессенджер, dev.max.ru)
**Статус:** MVP запущен 02.04.2026, бот отвечает, онбординг работает

---

## 2. Бизнес-модель

- **Free (30 дней):** AI-чат, до 3 фото еды/день, простые рекомендации
- **Premium (через QR):** покупатель Moonvit фотографирует QR-код под крышкой витаминов и отправляет фото в чат. Бот декодирует QR, продлевает подписку. Безлимитные фото, deepcheck, анализ лаб. результатов
- **Проблема QR:** текущие QR под крышкой ведут на Telegram-бота. Пользователь должен сфотографировать QR камерой в MAX (отправить как фото), а не сканировать

## 3. Технический стек

| Компонент | Технология |
|-----------|-----------|
| Backend | Node.js / TypeScript, Vercel serverless (fra1) |
| AI чат | OpenAI GPT-4.1-mini (temperature 0.7, max 1500 tokens) |

| Админка | React + Vite + Tailwind CSS, отдельный Vercel проект |
| Мессенджер API | MAX Bot API (platform-api.max.ru), webhook |

## 5. Текущий онбординг (12 шагов)

1. Согласие на обработку ПДн (152-ФЗ) - кнопки "Согласен" / "Отказаться"
2. Как тебя зовут? (текст)
3. Поделись номером телефона (кнопка request_contact или "пропустить")
4. Пол (кнопки М/Ж)
5. Возраст (число)
6. Рост в см (число)
7. Вес в кг (число)
8. Цель (кнопки: похудеть / поддержать / набрать)
9. Активность (кнопки: 5 уровней)
10. Аллергии (текст или "нет")
11. Хронические заболевания (текст или "нет")
12. Предпочтения питания (текст или "нет")

После онбординга: расчёт КБЖУ по формуле Mifflin-St Jeor и показ главного меню.

## 6. Текущие промпты (все редактируются из админки)

### Системный промпт чата
```
Ты - AI-нутрициолог бренда Moonvit. Помогаешь пользователям с питанием, подбором витаминов и здоровым образом жизни. Отвечай на русском. Будь дружелюбным и профессиональным. Сначала рекомендуй еду, потом витамины. Не давай медицинских назначений.
```

### Промпт Vision (фото еды)
```
Проанализируй фото еды. Определи блюда и продукты. Для каждого оцени: калории, белки, жиры, углеводы. Ответь JSON: {"items": [{"name": "...", "calories": N, "protein": N, "fat": N, "carbs": N}], "total": {"calories": N, "protein": N, "fat": N, "carbs": N}, "comment": "..."}
```

### Промпт Vision (анализы крови)
```
Проанализируй фото результатов анализов крови. Извлеки все показатели в JSON: {"markers": [{"name": "...", "value": "...", "unit": "...", "reference": "...", "status": "normal|low|high"}]}. Затем дай интерпретацию на русском.
```

### Промпты 4 агентов deepcheck
- Диетолог: анализ рациона, рекомендации по продуктам
- Здоровье: анализ лаб. результатов, дефициты
- Лайфстайл: активность, сон, стресс
- Составитель отчёта: объединяет 3 агента в один отчёт до 3500 символов

## 7. Текущий пользовательский путь

```
Пользователь открывает бота → "Начать" →
  Согласие ПДн → Онбординг (12 вопросов) →
    Главное меню (7 кнопок):
      📸 Фото еды → отправляет фото → бот считает КБЖУ → сохранить?
      🔬 Deepcheck → 4 AI-агента параллельно → отчёт
      🧪 Анализы → фото анализов → парсинг + интерпретация
      📊 Дневник → статистика за день
      💧 Вода → +1 стакан (трекер 0/8)
      🎁 QR-код → фото QR → активация Premium
      👤 Профиль → сводка профиля
```

Помимо кнопок, пользователь может писать свободный текст — бот ответит через AI-чат с учётом профиля и истории (контекст 10 последних сообщений + автоматическое резюмирование каждые 50 сообщений).

---

## 8. Ограничения платформы MAX

- Лимит текста сообщения: 4000 символов
- Inline-клавиатура: до 210 кнопок, до 30 рядов, до 7 в ряду
- Rate limit: 30 rps
- Webhook: только HTTPS
- Нет Web App / Mini App (как в Telegram)
- Нет форматирования rich cards — только текст + inline кнопки + изображения
- Документация расходится с реальными payload (нужно логировать всё)

## 9. Задание на исследование

Проведи глубокий анализ NutriBot по следующим направлениям. Для каждого направления дай:
- Оценку текущего состояния (1-10)
- Конкретные проблемы
- Конкретные решения с примерами текстов/промптов

### 9.1 UX и пользовательский путь

**Проблема:** Онбординг слишком длинный (12 вопросов до первого полезного действия). Пользователь теряет интерес.

Исследуй:
- Как сократить онбординг до 2-3 шагов максимум?
- Как показать ценность СРАЗУ на первом экране? (Например: "Отправь фото любой еды — я посчитаю калории за 5 секунд!")
- Какие данные собрать позже, в процессе общения?
- Как сделать "wow-эффект" в первые 30 секунд?
- Оптимальная последовательность: сначала демо-ценность, потом сбор данных

### 9.2 Свободная коммуникация vs кнопки

**Проблема:** Бот сейчас предлагает кнопки, но пользователь часто хочет написать текстом. Нужна гибкость.

Исследуй:
- Какие действия должны быть доступны и через кнопки, И через текст?
- Как бот должен понимать намерение из свободного текста? (NLU на уровне промпта?)
- Примеры: "хочу сфоткать еду" → переход в режим фото, "покажи мой дневник" → /today
- Как обрабатывать голосовые сообщения (если MAX поддерживает)?
- Баланс: когда показывать кнопки, когда нет?

### 9.3 Тон и характер бота

Исследуй:
- Каким должен быть характер NutriBot? (друг, тренер, эксперт, мотиватор?)
- Примеры приветственных сообщений (5 вариантов разного тона)
- Как бот должен реагировать на: вредную еду, срыв диеты, достижение цели, долгое отсутствие?
- Эмодзи: сколько, какие, когда уместны?
- Как бот обращается к пользователю (на ты/вы, по имени?)

### 9.4 Система напоминаний

Исследуй:
- Какие напоминания нужны? (утреннее приветствие, напоминание о воде, вечерний итог дня)
- Как пользователь включает/отключает напоминания?
- Можно ли технически отправлять сообщения по расписанию через MAX API? (проактивные сообщения)
- Тексты напоминаний (не раздражающие, полезные)
- Частота: ежедневно? Через день? По дням активности?

### 9.5 Визуал и медиа

**Проблема:** Бот сейчас отвечает только текстом. Нет картинок, инфографики, прогресс-баров.

Исследуй:
- Можно ли в MAX отправлять изображения ботом? Как?
- Идеи визуального контента: инфографика КБЖУ, прогресс-бар дня, рецепты с фото
- Можно ли генерировать простые chart-изображения на лету (canvas → png → upload)?
- Стоит ли добавить фото продуктов Moonvit в рекомендации?

### 9.6 Улучшение промптов

Исследуй:
- Системный промпт: как сделать бота более "живым" и менее "AI-шаблонным"?
- Промпт Vision (еда): как повысить точность распознавания? Учёт размера порции?
- Промпт deepcheck: как сделать отчёт полезнее?
- Интеграция knowledge base витаминов в промпт — как рекомендовать нативно?
- Промпт для ситуаций: "я на диете но хочу пиццу", "у меня аллергия на X, что можно?"

### 9.7 Монетизация и growth

Исследуй:
- Оптимальная длина бесплатного периода (сейчас 30 дней — не много ли?)
- Что должно быть в free, что в premium?
- Как стимулировать покупку витаминов через бот (без навязчивости)?
- Реферальная программа: "Приведи друга → +7 дней premium"
- Виральные механики: шаринг результатов, челленджи

### 9.8 Конкурентный анализ

Исследуй:
- Какие аналоги есть на рынке? (FatSecret, MyFitnessPal, Yazio, боты в Telegram)
- В чём NutriBot сильнее? В чём слабее?
- Какие фичи есть у конкурентов, которых нет у NutriBot?
- Уникальное преимущество: AI-нутрициолог + витамины Moonvit + MAX (нет конкурентов в MAX!)

### 9.9 Рекомендации витаминов

Исследуй:
- Как нативно рекомендовать витамины? (не "купи Moonvit", а "тебе может не хватать витамина D, вот что поможет...")
- В каких ситуациях рекомендация уместна?
- Как связать с дефицитами из анализов?
- Talking points для каждого SKU (что говорить, что НЕ говорить)
- Частота упоминания (сейчас: не чаще 1 раза на 5 сообщений)

### 9.10 Техническая устойчивость

Исследуй:
- Vercel serverless: timeout 10 сек — достаточно ли для Vision API? Нужен ли VPS?
- Что если OpenAI API упадёт? Fallback-ответы?
- Rate limiting: 30 rps для MAX — как обрабатывать пики?
- Стоимость: сколько стоит один пользователь в месяц (OpenAI tokens)?
- Мониторинг: как отслеживать ошибки, задержки, расход токенов?
